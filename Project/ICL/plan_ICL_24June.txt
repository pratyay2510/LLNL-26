Research Plan — Retrieval-Augmented Steering (RAS): Query-Conditional Latent Control for Reasoning
Context
The user is converging on a specific, sharper thesis than the earlier "Fisher-direction steering": make latent steering query-aware / prompt-aware instead of applying one global vector to every input. The intuition is RAG-like: the prompt is a signal that retrieves which steering vector(s) to pull from a corpus/library of demonstration-derived vectors, composes them, and injects them at certain layers — at negligible cost. The canonical example: a math question retrieves a math-reasoning steering vector (not a global average), steering toward better reasoning. Final goal = improving prompt-aware reasoning. The user wants (1) a fine-toothed literature review with a prioritized reading list, (2) a concrete, novel, defensible problem statement (not a solution), (3) an honest novelty verdict, (4) all sources, (5) a solid research plan. No overclaiming, no infeasible rabbit holes, efficiency is paramount.

1. Novelty verdict (BLUF — honest, not validating)
"Query-conditional / prompt-aware steering" is NOT novel. It is an active, crowded 2025–26 area. If the pitch is just "make steering input-dependent," it is already done many ways:

Gating a single vector by an input condition: CAST (ICLR'25), K-CAST (AAAI'25, for reasoning).
Generating a vector from the input (parametric): HyperSteer (hypernetwork), SADI (ICLR'25, element-wise), Guiding Giants (controller), SVF/FLAS (velocity fields), CLAS (context-dependent coefficients).
Per-instance projection: PDS (Prototype-Based Dynamic Steering — projects the query onto reasoning prototypes; targets GSM8K).
Composing a fixed basis: Steer2Adapt (few-shot calibration), Conceptors, MAT-Steer (multi-attribute).
Static library (no router): EasySteer (precomputed vectors for 8 domains, manual selection).
What is genuinely open (verified across two independent searches + my own checks of the 4 closest works): a non-parametric, extensible, Retrieval-Augmented Steering system, i.e. a query → retrieve-from-a-growing- bank-of-many-skill-vectors → compose → inject pipeline, with an uncertainty-gated "do no harm" fallback, focused on reasoning, at near-zero cost (a vector add, no extra context tokens, no retraining to add a skill). Precise differentiators from the four closest competitors:

Closest work	What it does	Why it is NOT this	Verified?
HyperSteer (2506.03292)	trained hypernetwork generates a vector from a steering prompt	parametric generation, not retrieval; new skill ⇒ relies on hypernet generalization/retraining, not a pluggable bank	✅ fetched
Steer2Adapt (2602.07276)	composes a fixed low-dim basis via few-shot calibration per task	bounded fixed subspace; adapts per task with calibration examples, not per query zero-shot retrieval from a growing bank	✅ fetched

So the defensible novelty is the system + the routing intelligence + the no-harm guarantee + reasoning, not "query-dependent steering" in the abstract. The wedge is the RAG analogy made real for activation steering: just as RAG decouples knowledge from weights (retrieve documents at inference), RAS decouples skills from weights (retrieve steering vectors at inference) — but unlike demonstration-RAG it costs ~0 context tokens, and unlike LoRA-routing it needs no weight merge or training. This framing is, to our knowledge, unoccupied.

Reviewer risk to respect: the space is dense and moving fast; "yet another conditional-steering paper" is the rejection. Mitigations baked into the problem below: (i) the extensibility/RAG property no competitor has, (ii) the no-harm uncertainty gate (turns the documented unreliability of steering into a feature), (iii) the heterogeneity thesis (where global steering provably fails), (iv) reasoning focus + breadth of eval.

2. Concrete problem statement (the deliverable)
Motivation. Activation steering can improve reasoning but is documented to be unreliable and input- dependent (Tan, NeurIPS'24; Braun '26), and even "benign" steering can cause harmful side-effects (Steering-Externalities '26). A global vector applied to every query is a compromise average that helps some inputs and silently degrades others — unacceptable for deployment. The deepest unmet need is not "steer harder" but steer only when it is safe to, and abstain otherwise.

Problem. Can query-conditional latent steering for reasoning be equipped with a calibrated abstention gate that controls the rate of harm — the probability that steering flips a baseline-correct answer to wrong — at a user-chosen level α via conformal risk control, while still capturing the upside when retrieval is confident? Concretely: build an open-domain steering bank by clustering a large demonstration corpus into auto-discovered skills (one vector per cluster, offline); per query, retrieve the nearest skill vector(s), compose them, and steer-or-abstain by a conformally-calibrated confidence score — all at negligible cost (query embedding + cosine lookup + vector add; zero extra context tokens; new skills = new bank entries, no retraining).

The guarantee (stated honestly — not overclaimed). NOT "never harms any instance." Rather: under exchangeable calibration data, conformal risk control bounds the expected harm rate ≤ α with high probability, trading coverage (how often we steer) for safety. The headline result is the risk–coverage frontier: how much reasoning gain is retained at a guaranteed harm rate — a control knob global steering does not have.

Success criteria (falsifiable): (1) at a fixed guaranteed α, RAS-with-gate gives net reasoning gains over no-steering on a heterogeneous mixture, while global steering violates α (harms a measurable fraction); (2) the empirical harm rate tracks the conformal target α across α; (3) extensibility — adding bank entries for a new corpus slice helps those queries with no regression elsewhere; (4) negligible overhead vs demonstration-RAG (token + latency accounting).

The math-question example, made concrete: query → router scores it as "arithmetic/algebra" → retrieves the math-reasoning bank vector(s) (built offline from math demonstrations, optionally with the Fisher/FRE estimator of §5) → composes with any secondary skill → injects at the input-selected layer → if the router is unsure, abstain (no steering). Negligible overhead: one query embedding + a cosine lookup + a vector add.

Why top-tier: first retrieval-augmented, extensible, no-harm steering framework; reframes steering's known unreliability as a routing + abstention problem; clean RAG analogy; strong efficiency. Why 1-month-tractable: bank = clustered CAA/ICV vectors (public code); router = cosine-kNN to start (trivial); infra reusable (EasySteer, pyreft, ICV); the headline experiment (heterogeneity) is a small mixed-benchmark eval.

3. Intuitive + mathematical approaches to explore (design space, not a committed solution)
A. Bank construction (offline, cheap). Cluster a demonstration corpus into skills (k-means on query embeddings, or use dataset/task labels). Per cluster c, compute a steering vector v_c via CAA mean-diff, ICV (PCA of h(y)−h(x)), or the Fisher/FRE estimator of §5 (variance-aware → more reliable per-vector). Store (key_c, v_c, layer_c, σ_c) where key_c = cluster centroid in an embedding space, σ_c = a reliability/spread score.

B. Routing (per query, the core algorithmic novelty — several options to compare):

Embedding kNN: route by cosine(query embedding, bank keys); soft top-k weights w_c = softmax(sim/τ).
Model-internal routing: use the LLM's own hidden state at an early layer as the query key (cheaper, no external encoder; ties routing to the space being steered).
Uncertainty/confidence routing: compute predictive entropy; high entropy ⇒ steer (intervene where the model is unsure), low entropy ⇒ leave alone. Decides when.
Interpretability routing: map the query to the SAE feature(s) it activates, retrieve the vector indexed by that feature. Decides which, interpretably. C. Composition (when multiple skills fire). z = h + Σ_c w_c · v_c, but naive linear sums interfere (documented). Use orthogonalization / subspace allocation (MAT-Steer, MSRS) or gated/non-linear composition (K-Steering). Keep top-k small (k≤2–3). D. Injection site. Single middle layer to start; optionally input-dependent layer (à la "Where to Steer"). E. No-harm gate (headline safety property). Define a confidence ρ(query) (router margin and/or model entropy). If ρ below threshold, abstain (apply nothing). Target a guarantee: RAS ≥ baseline on every sub-task. This directly answers the "steering can hurt" critique (Tan/Braun). F. Math framing to lean on. Routing = a non-parametric mixture-of-steering-experts (kernel/Nadaraya– Watson weights over bank keys); composition = constrained vector addition under orthogonality; gating = a selective-prediction / risk-coverage problem (abstain to bound risk). These give the paper rigor without new training.
4. Research plan (de-risked, ~4 weeks, efficiency-first)
Phase 0 — open-domain bank (days 1–5): pick a reasoning-heavy instruction corpus; embed queries; cluster (k-means, start k≈50–100, scale later) → one CAA/ICV (optionally Fisher/FRE §5) vector per cluster + key + reliability score. Reproduce one CAA + one PDS number (gate). Tractability caution: start with a moderate corpus and k; the bank is the riskiest engineering — timebox it. Reuse EasySteer/pyreft/ICV.
Phase A — the headline: conformal no-harm gate (days 6–14). Define harm = baseline-correct → steered- wrong. Build a confidence score from router margin + model signal (not entropy alone — see 2603.21172); calibrate the steer/abstain threshold by conformal risk control on held-out calibration data. Deliver the risk–coverage frontier: RAS-gate vs global-steering vs no-steer on a heterogeneous mixture. Show empirical harm rate ≤ α (tracks the conformal target) while global steering violates α. This is the paper.
Phase B — routing & composition (days 15–21): compare router signals (embedding-kNN vs model-internal-state vs SAE-feature); multi-skill queries → composition (orthogonalized vs naive, top-k≤3); ablate bank granularity k. Goal: maximize coverage at fixed α.
Phase C — extensibility & generalization (days 22–26): add a new corpus slice → new bank entries, no retraining → gain on those queries, no regression elsewhere + α still held; unseen-but-related routing; Fisher/FRE-bank vs CAA-bank ablation.
Write-up (days 27–30): ≥5 seeds, cost accounting (tokens=0 vs demo-RAG; latency), draft.
Baselines: no-steer · best global CAA/ICV · PDS · K-CAST · demonstration-RAG (EPR/kNN-prompting, to expose the context-token cost) · oracle routing (skyline) · ungated RAS (to show the gate's value). Models: Llama-3.1-8B / Qwen2.5-7B (+ a reasoning-distilled model). Metrics: empirical-harm-rate@α and coverage (the headline risk–coverage curve); per-skill & aggregate accuracy; routing accuracy; cost (extra tokens, latency).

5. The earlier Fisher/FRE result — folded in as a component (not the headline)
From the prior literature pass (verified): standard directions use class means (CAA/RepE) or top-PCA (ICV) and ignore within-class covariance; the Fisher/LDA direction Σ_W⁻¹(μ⁺−μ⁻) (diagonal = "FRE" 1/σ weighting) is unexplored as a steering method (SEA is spectral covariance-max, not Fisher; Braun uses LDA only as a diagnostic). In RAS, this is the per-bank-vector estimator: build each v_c with Fisher/FRE for higher reliability. It is a clean ablation ("does a variance-aware bank beat a mean-diff bank?"), not the main claim.

Crowded space → lead with the RAG/extensibility + no-harm + heterogeneity wedge, not "conditional steering."
Router errors hurt → the no-harm uncertainty gate is not optional; it is the safety contribution.
Composition interference → keep top-k ≤ 2–3; use orthogonalization; ablate naive vs orthogonal.
"Just MoE/LoRA-routing" critique → emphasize: activation-space, training-free, no weight merge, 0 context tokens, pluggable bank. Quantify the cost gap vs demonstration-RAG.
Gains only under heterogeneity → that's the point; state the scope honestly (single-task ⇒ RAS≈global).
Don't rebuild the dropped inversion/on-manifold machinery; don't over-engineer a learned router before the cosine-kNN baseline proves the heterogeneity thesis.
9. Resolved decisions (user, 2026-06-23)
Headline = the conformal no-harm guarantee. The paper leads with: a calibrated steer-or-abstain gate that controls the harm rate ≤ α (conformal risk control) — the risk–coverage frontier is the money figure. Novelty rests here (conformal abstention applied to the steering decision is open); the bank/retrieval is the setting. Working name: "Conformal Steering" / "Safe RAS" — NOT "Selective Steering" (taken, 2601.19375).
Bank scope = open-domain corpus-clustered. Ambitious; it makes routing messy, which is precisely what makes the no-harm gate necessary and impressive. Biggest timeline risk — mitigate by timeboxing Phase 0, starting with a moderate corpus and k≈50–100, scaling only if Phase A succeeds.
Honest-guarantee reminder for the write-up: claim expected harm-rate control at level α w.h.p. under exchangeability (conformal), trading coverage — NOT "never harms." Overclaiming here is the fastest path to desk-rejection.
